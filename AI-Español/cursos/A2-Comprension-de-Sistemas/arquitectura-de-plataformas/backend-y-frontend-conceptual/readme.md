# 🖥 Backend y frontend (conceptual)  
### La diferencia entre lo que ves y lo que sostiene el sistema

Cuando abres una app o una página web, interactúas con una capa visible.

Esa capa se conoce como **frontend**.

Es la parte con la que haces clic, escribes, navegas y lees.

Pero esa capa no funciona sola.

Detrás existe otra parte menos visible: el **backend**.

---

# 🧠 Qué hace cada capa

De forma conceptual, podemos pensarlo así.

## Frontend
Es la interfaz.

Muestra menús, publicaciones, botones, formularios y resultados.

## Backend
Es la parte que procesa lo que ocurre detrás.

Ahí viven cosas como:

- bases de datos  
- reglas del sistema  
- gestión de cuentas  
- almacenamiento  
- lógica de negocio  

El usuario ve el frente.  
El sistema opera desde atrás.

---

# ⚙ Por qué importa esta diferencia

A veces una plataforma parece simple porque su interfaz se siente simple.

Pero esa apariencia puede ocultar una estructura compleja.

Cada acción visible —dar like, iniciar sesión, publicar algo— suele depender de procesos invisibles que:

- verifican permisos  
- guardan datos  
- aplican reglas  
- activan otras funciones  

Entender esto ayuda a no confundir experiencia de uso con estructura real del sistema.

---

# 🌐 Lo visible no agota la explicación

Muchas de las decisiones más importantes no se toman en el frontend.

Se toman en el backend.

Ahí se define, por ejemplo:

- qué datos se guardan  
- cuánto tiempo se conservan  
- cómo se ordena un feed  
- qué contenido activa alertas o restricciones  

Por eso la capa visible puede dar una sensación engañosa de transparencia.

---

# ⚡ Una idea importante

En el entorno digital, lo que ves no es todo lo que existe.

A veces ni siquiera es la parte más importante.

Mirar la diferencia entre frontend y backend ayuda a entender dónde ocurre buena parte del poder operativo de una plataforma.
